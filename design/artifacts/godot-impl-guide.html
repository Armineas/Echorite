Configuration du projet

Prérequis absolu. Sans ça, rien ne compile.

1 — Vérifier les Autoloads

Project › Project Settings › Globals › Autoload. Les 6 entrées suivantes doivent être présentes et dans cet ordre exact (l'ordre détermine les dépendances) :

NomCheminRôle
GSMres://src/autoload/GameStateManager.gdSource de vérité de l'état de jeu
SaveSystemres://src/autoload/SaveSystem.gdPersistence sur disque
SceneSequencerres://src/autoload/SceneSequencer.gdOrdre chronologique des scènes
SceneRouterres://src/autoload/SceneRouter.gdNavigation entre scènes
DialogueEngineres://src/core/dialogue/DialogueEngine.gdMoteur de dialogue
CompanionRegistryres://src/core/companions/CompanionRegistry.gdDonnées des compagnons
Ordre critique

SceneSequencer doit être avant SceneRouter dans la liste. SceneRouter appelle SceneSequencer.get_next() dans son advance_scene(). Si l'ordre est inversé, SceneSequencer sera null lors de l'appel.

2 — Scène principale

  1. Project › Project Settings › Application › Run › Main Scene → pointer vers res://src/ui/MainMenu.tscn (à créer en P0).

3 — Input Map

Project › Project Settings › Input Map. Ajouter ces actions :

ActionTouchesUsage
dialogue_advanceSpace, Enter, Left ClickAvancer dans le dialogue
dialogue_skipCtrl + SpacePasser le texte en cours (à implémenter)
ui_cancelEscapePause menu (existant par défaut)

4 — Paramètres d'affichage

  1. Project Settings › Display › Window : Width = 1280, Height = 720.
  2. Stretch Mode = canvas_items, Aspect = keep. Cela permet au jeu de s'adapter aux résolutions sans déformer.
  3. Test Width / Test Height = 1280 × 720 pour le dev.

DialogueUI.tscn

Débloque : afficher n'importe quel dialogue, description, ou choix. Composant central instancié dans chaque scène personnage.

Principe

DialogueUI se connecte aux signaux de DialogueEngine et affiche le bon container selon le type de nœud (line / description / choice). C'est une scène autonome instanciée comme enfant de chaque scène personnage.

Structure de nœuds

DialogueUI Control — Anchor : Full Rect │ ├── DescriptionBox PanelContainer — ancré haut, centré │ └── MarginContainer │ └── description_text RichTextLabel bbcode_enabled=true, fit_content=true │ ├── DialogueBox PanelContainer — ancré bas, pleine largeur │ └── HBoxContainer │ ├── portrait TextureRect custom_minimum_size=(80,80), stretch_mode=keep_aspect_centered │ └── VBoxContainer │ ├── speaker_name Label theme_override_font_size=13 │ └── dialogue_text RichTextLabel bbcode_enabled=true, scroll_active=false, fit_content=true │ ├── ChoiceContainer VBoxContainer — ancré bas-droite │ └── (boutons ajoutés dynamiquement via GDScript) │ └── ContinueHint Label text="[ Espace ]", anchors bas-droit

DialogueUI.gd

res://src/ui/dialogue/DialogueUI.gd
extends Control

## DialogueUI — Composant d'affichage du dialogue.
## À instancier comme enfant de chaque scène personnage.
## Se connecte aux signaux de DialogueEngine au _ready().

@onready var description_box: PanelContainer = $DescriptionBox
@onready var description_text: RichTextLabel = $DescriptionBox/MarginContainer/description_text
@onready var dialogue_box: PanelContainer = $DialogueBox
@onready var portrait: TextureRect = $DialogueBox/HBoxContainer/portrait
@onready var speaker_name: Label = $DialogueBox/HBoxContainer/VBoxContainer/speaker_name
@onready var dialogue_text: RichTextLabel = $DialogueBox/HBoxContainer/VBoxContainer/dialogue_text
@onready var choice_container: VBoxContainer = $ChoiceContainer
@onready var continue_hint: Label = $ContinueHint

# Portraits par personnage (à remplir quand les assets sont prêts)
const PORTRAITS: Dictionary = {
    "Aldric": null,
    "Seïra": null,
    "Varek": null,
    "Daven": null,
}

func _ready() -> void:
    DialogueEngine.line_displayed.connect(_on_line_displayed)
    DialogueEngine.description_displayed.connect(_on_description_displayed)
    DialogueEngine.choice_presented.connect(_on_choice_presented)
    DialogueEngine.dialogue_ended.connect(_on_dialogue_ended)
    hide()

func _input(event: InputEvent) -> void:
    if not DialogueEngine.is_running():
        return
    if event.is_action_pressed("dialogue_advance"):
        DialogueEngine.advance()
        get_viewport().set_input_as_handled()

func _on_line_displayed(speaker: String, text: String, is_internal: bool) -> void:
    _hide_all()
    show()
    dialogue_box.show()
    speaker_name.text = speaker
    if is_internal:
        # Monologue intérieur : italique, teinte froide
        dialogue_text.text = "[i][color=#a8bfd4]" + text + "[/color][/i]"
        speaker_name.add_theme_color_override("font_color", Color(0.6, 0.75, 0.9))
    else:
        dialogue_text.text = text
        speaker_name.remove_theme_color_override("font_color")
    # Portrait
    var tex: Texture2D = PORTRAITS.get(speaker, null)
    portrait.texture = tex
    portrait.visible = tex != null
    continue_hint.show()

func _on_description_displayed(text: String) -> void:
    _hide_all()
    show()
    description_box.show()
    # Intertitle centré, italique — comme un carton de film
    description_text.text = "[center][i]" + text + "[/i][/center]"
    # Avance automatiquement sur input (pas de choix à faire)
    continue_hint.show()

func _on_choice_presented(choices: Array) -> void:
    _hide_all()
    show()
    choice_container.show()
    # Nettoyer les anciens boutons
    for child in choice_container.get_children():
        child.queue_free()
    # Créer un bouton par choix visible
    for i in choices.size():
        var btn := Button.new()
        btn.text = choices[i]["text"]
        btn.focus_mode = Control.FOCUS_ALL
        var idx: int = choices[i]["index"]
        btn.pressed.connect(func(): DialogueEngine.select_choice(idx))
        choice_container.add_child(btn)
    # Focus automatique sur le premier choix (accessibilité clavier)
    await get_tree().process_frame
    if choice_container.get_child_count() > 0:
        choice_container.get_child(0).grab_focus()

func _on_dialogue_ended(_id: String) -> void:
    hide()
    # SceneRouter avance vers la prochaine scène de la timeline
    SceneRouter.advance_scene()

func _hide_all() -> void:
    description_box.hide()
    dialogue_box.hide()
    choice_container.hide()
    continue_hint.hide()

Étapes dans l'éditeur

  1. Créer le dossier res://src/ui/dialogue/.
  2. Créer une nouvelle scène de type Control, nommer le nœud racine DialogueUI, sauvegarder en DialogueUI.tscn.
  3. Sélectionner DialogueUI, mettre les 4 anchors à Full Rect (Layout › Full Rect).
  4. Ajouter DescriptionBox (PanelContainer) : ancré haut-centre, largeur 80% de l'écran, centré. Ajouter MarginContainer dedans (marges 16px), puis RichTextLabel nommé description_text.
  5. Ajouter DialogueBox (PanelContainer) : ancré bas, pleine largeur, hauteur 180px. Ajouter HBoxContainer + TextureRect (portrait, 80×80) + VBoxContainer › Label (speaker_name) + RichTextLabel (dialogue_text).
  6. Ajouter ChoiceContainer (VBoxContainer) : ancré bas-droite, largeur 400px. Laisser vide (rempli dynamiquement).
  7. Ajouter ContinueHint (Label) : texte [ Espace ], ancré bas-droit, couleur grisée.
  8. Attacher DialogueUI.gd au nœud racine.
  9. Activer bbcode_enabled = true sur les deux RichTextLabel.
Test à ce stade

Depuis l'éditeur, dans le tab Output, appeler via la console de debug : DialogueEngine.start("acte1/aldric_a1.dialogue.json", "aldric_a1_s1_frontiere"). Les signaux devraient déclencher l'affichage si DialogueUI est dans la scène active.

res://src/ui/TimelineView.gd
extends Control

@onready var act_label: Label = $CenterContainer/VBoxContainer/act_label
@onready var scene_name: Label = $CenterContainer/VBoxContainer/scene_name
@onready var continue_btn: Button = $CenterContainer/VBoxContainer/continue_btn

var _pending_entry: Dictionary = {}

func _ready() -> void:
    var act := SceneSequencer.current_act()
    act_label.text = "ACTE %d" % act
    var next := SceneSequencer.get_next()
    if not next.is_empty():
        scene_name.text = next.get("seq_id", "…")
    continue_btn.pressed.connect(_on_continue)
    continue_btn.grab_focus()

func _receive_scene_entry(entry: Dictionary) -> void:
    # Appelé par SceneRouter lors d'une transition d'acte
    if entry.get("act_transition", false):
        act_label.text = "FIN DE L'ACTE %d" % entry.get("from_act", 1)
        scene_name.text = "Acte %d — À venir" % entry.get("to_act", 2)

func _on_continue() -> void:
    SceneRouter.advance_scene()

res://src/ui/MainMenu.gd
extends Control

@onready var new_game_btn: Button = $ButtonColumn/new_game_btn
@onready var continue_btn: Button = $ButtonColumn/continue_btn
@onready var quit_btn: Button = $ButtonColumn/quit_btn

func _ready() -> void:
    new_game_btn.pressed.connect(_on_new_game)
    continue_btn.pressed.connect(_on_continue)
    quit_btn.pressed.connect(_on_quit)
    # Désactiver "Continuer" si pas de sauvegarde

    SceneRouter.advance_scene()

func _on_quit() -> void:
    get_tree().quit()

extends Node2D

@onready var background: TextureRect = $Background

# Fonds par scène — à remplir avec les assets quand disponibles
const SCENE_BACKGROUNDS: Dictionary = {
    "aldric_a1_s1_frontiere": "res://assets/backgrounds/frontiere.png",
    "aldric_a1_s2_nira":      "res://assets/backgrounds/transit.png",
    "aldric_a1_s3_reception": "res://assets/backgrounds/quartier_general.png",
    "aldric_a1_s4_brennan":   "res://assets/backgrounds/auberge.png",
    "aldric_a1_cp1":          "res://assets/backgrounds/place_principale.png",
}

func _receive_scene_entry(entry: Dictionary) -> void:
    var scene_id: String = entry.get("scene_id", "")
    _set_background(scene_id)
    DialogueEngine.start(
        entry.get("dialogue_path", ""),
        scene_id,
        entry.get("entry_node", "start")
    )

func _set_background(scene_id: String) -> void:
    if not SCENE_BACKGROUNDS.has(scene_id):
        return
    var path: String = SCENE_BACKGROUNDS[scene_id]
    if ResourceLoader.exists(path):
        background.texture = load(path)

Transitions & fondus

Débloque : changements de personnage non-brutaux. Fondu noir entre chaque scène.

Approche recommandée : CanvasLayer overlay

Ajouter un CanvasLayer (layer = 100) dans chaque scène de jeu, contenant un ColorRect Full Rect noir, modulate.a = 0 au départ. Un AnimationPlayer gère le fade.

Alternativement, SceneRouter peut gérer le fade globalement via un CanvasLayer qu'il instancie une seule fois. C'est plus propre :

À ajouter dans SceneRouter.gd
## Dans _ready() de SceneRouter :
var _fade_overlay: ColorRect
var _canvas_layer: CanvasLayer

func _ready() -> void:
    _canvas_layer = CanvasLayer.new()
    _canvas_layer.layer = 100
    add_child(_canvas_layer)
    _fade_overlay = ColorRect.new()
    _fade_overlay.anchor_right = 1.0
    _fade_overlay.anchor_bottom = 1.0
    _fade_overlay.color = Color.BLACK
    _fade_overlay.modulate.a = 0.0
    _canvas_layer.add_child(_fade_overlay)

## Remplacer le bloc _change_scene par :
func _change_scene(path: String, entry: Dictionary = {}) -> void:
    _transition_in_progress = true
    # Fondu au noir
    var tw := create_tween()
    tw.tween_property(_fade_overlay, "modulate:a", 1.0, 0.4)
    await tw.finished
    # Changement de scène
    get_tree().change_scene_to_file(path)
    await get_tree().process_frame
    # Transmettre l'entrée
    if not entry.is_empty():
        await get_tree().process_frame
        var root := get_tree().current_scene
        if root != null and root.has_method("_receive_scene_entry"):
            root._receive_scene_entry(entry)
    # Fondu au clair
    tw = create_tween()
    tw.tween_property(_fade_overlay, "modulate:a", 0.0, 0.4)
    await tw.finished
    _transition_in_progress = false
    transition_ended.emit()
    scene_changed.emit(_current_character, path)

Système de combat — Aldric

Mode RPG tour par tour. À activer quand le dialogue d'une scène de combat se termine.

ALDRIC

Architecture

Le combat se déclenche quand un nœud de dialogue de type "auto" ou "choice" a un effet start_combat (à ajouter à DialogueEngine). La scène AldricWorld affiche alors le GameplayLayer (masqué pendant le dialogue).

  1. Créer res://src/modes/aldric/combat/CombatSystem.gd — gère les tours, les unités, les actions.
  2. Créer res://src/modes/aldric/combat/CombatUI.tscn — barres de vie, liste d'actions, ordre des tours.
  3. Les données de combat des compagnons viennent de CompanionData.create_combat_unit(act) (déjà écrit).
  4. Signal de fin de combat → retour au dialogue ou SceneRouter.advance_scene() selon le résultat.
  5. Résultat du combat → flag GSM (ALDRIC_WON_SKIRMISH, etc.) pour influencer les dialogues suivants.
Approche minimale pour débuter

Le combat peut d'abord être simulé : un bouton "Résoudre le combat" avec un résultat fixe, pour valider le flux avant d'implémenter le vrai système tour par tour.

Couverture, patrouilles, actions silencieuses.

DAVEN

Composants à créer

  1. BuildingMap.tscn — vue top-down du bâtiment administratif. Couloirs, portes, positions des agents SI. Rendu très sombre, lumière uniquement aux lampes.
  2. CoverMeter.tscn — indicateur de couverture (0–100%). Baisse si Daven agit trop vite ou fait des erreurs. Visible en permanence dans un coin.
  3. PatrolSystem.gd — les agents de la Surveillance Interne bougent sur des chemins prédéfinis. NavigationAgent2D ou patterns fixes selon la complexité voulue.
  4. Les choix dans les dialogues de Daven peuvent modifier le CoverMeter via des effets custom à ajouter à DialogueEngine ("type": "modify_cover", "value": -10).
